## Conventions observed

- **Layering**: screens → services → models (screens never import other screens directly)
- **Naming**: snake_case files, PascalCase classes; service files suffixed `_service`
- **Firebase**: all Firestore reads go through a service; screens do not call Firebase directly
- **New screen**: add file to `lib/screens/`, wire route in `app.dart`
- **New service**: add file to `lib/services/`, inject into screens via constructor or provider
- **New model**: add to `lib/models/`
